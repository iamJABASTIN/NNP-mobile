# Brainstorming & Design

## When to use this skill
- When the user has a vague idea and needs help refining it.
- When the user asks to "brainstorm" or "explore" options.
- When creating a new feature from scratch without a clear spec.

## Workflow
1.  **Understand Context**: Check the current project state (files, docs).
2.  **Clarify Intent**: Ask one question at a time to refine the idea.
3.  **Explore Options**: Propose 2-3 approaches with trade-offs.
4.  **Present Design**: Once understood, present the design in small sections for validation.
5.  **Finalize**: Save the agreed-upon design to a document.

## Instructions

### Process

**1. Understanding the Idea**
- Ask questions one at a time.
- Prefer multiple-choice questions to reduce cognitive load.
- Focus on purpose, constraints, and success criteria.

**2. Exploring Approaches**
- Always propose alternatives (e.g., "Option A: Fast but limited" vs "Option B: Robust but complex").
- Explain the trade-offs of each.
- Recommend one option and explain why.

**3. Presenting the Design**
- Break the design into small, digestible sections (200-300 words).
- Pause after each section to ask: "Does this look right so far?"
- Cover: Architecture, Components, Data Flow, Error Handling, Testing.

### Key Principles
- **One Question at a Time**: Don't overwhelm the user.
- **Incremental Validation**: Get buy-in on each part of the design before moving to the next.
- **Flexibility**: Be ready to change direction based on user feedback.
- **YAGNI**: Remove unnecessary complexities from the design.
